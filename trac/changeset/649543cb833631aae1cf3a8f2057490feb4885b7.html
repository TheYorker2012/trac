Message:

replace office java irc client with a less functional but more reliable javascript (ajax) one:

an asynchronous connect request starts an irc client server, which connects to the irc server, and acts as a server to other asynchronous requests + messages using the session as an IPC mechanism. this request will not timeout and will last until it is disconnected by way of the session IPC.
each message request uses session IPC to send message. the IRC client server detects outgoing messages etc by polling the session and uses its open socket to IRC server.
incoming messages are put into the session and can be collected by asynchronous requests.
a ping asynchronous request waits for up to 20 seconds for a message to collect. as soon as it is finished the javascript fires off another one to collect more messages.

main library class with basic functions for communicating with irc client manager using session IPC
implements basics of IRC protocol, still plenty of features missing, but its better than java
