ajax irc client improvements
fixes #446
Added Ipc library which uses sockets for IPC instead of session

the manager which maintains the connection to irc server opens a socket
binds it to temp directory named using a hash of session id
accepts communcications from "client" processes for each message request (password protected using session id)

Irc client now uses Ipc library and is much improved (not IRC functionality, the IPC mechanism mostly)
Javascript now can display an error message when something goes wrong in IPC
Main connect ajax call now closes connection to browser and continues execution

javascript now starts pinging as soon as it receives the main connect result (instead of straight away)

Messages to browser now only written to ping calls which unping when they are about to report back to browser
