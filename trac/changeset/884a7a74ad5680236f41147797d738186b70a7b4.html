Message:

#463 Comment editing (note there are changes to database and page properties accompanying this change, see ticket)
functional

the history of a comment is displayed in the comment smaller (for which there is a new style)
deleting of comments now requires confirmation (post data)
deleting of comments can now be done by the owner of a comment (but not undeletion)
comments can now be edited by the owner (technically also the moderator, though links aren't in place for this)
moderator deleted comments only shows deleted comments not deleted by the author of the comment

technical

added a comment_edits table and a couple of fields to comments to keep track of when and who made deletions
comments_model makes use of new tables/fields, and also has an edit function
rearranged CommentViewAdd? class in Comment_views to handle edits as well as adds (so as to share common code)
various changes to comments controller to accomodate the above functional changes/additions, and to share the "confirm with user" functionality and the "owner or moderator" permissions.
moved views/comment/report to a more general views/comment/confirm for confirming comment changes (to be used by delete as well).
also fixed a minor defect during preview where the box with wikitext didn't have escaped content. nothing critical.
